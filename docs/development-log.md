## 개발 규칙

### 코드 변경 로깅

1. 코드 변경 기록
   - 모든 코드 변경 사항이나 새로운 코드 생성 시 본 파일에 기록
   - 변경 날짜, 변경 내용, 영향받는 파일들을 상세히 기록
   - 변경 사유와 주요 로직 변경 사항 포함

2. 로그 형식
   - 날짜 (YYYY-MM-DD)
   - 변경 카테고리 (신규 기능, 버그 수정, 성능 개선 등)
   - 상세 변경 내용
   - 관련 파일 목록
   - 테스트 결과 (해당되는 경우)

3. 로그 위치
   - 모든 개발 로그는 `D:\APP\CorpEasy\docs\development-log.md`에 기록
   - 최신 변경 사항을 항상 파일 상단에 추가
